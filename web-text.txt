3.4 Разработка клиентской части
Согласно техническому заданию, клиентская часть состоит из web-интерфейса и Telegram чат-бота.
3.4.1 Разработка web-интерфейса приложения
Web-интерфейс приложения состоит из страницы авторизации и главной страницы приложения. Главная страница в свою очередь состоит из вкладок с задачами и со статистикой.
3.4.1.1 Страница авторизации
Если у пользователя не выполнен вход в приложение, то он попадает на страницу авторизации. На данной странице пользователь видит приветственное окно с описанием основных функций и особенностей сайта. Также здесь находится кнопка входа через Telegram. 
Интерфейс страницы авторизации представлен на рисунке 3.4.1
 
Рисунок 3.4.1 – Интерфейс страницы авторизации
	После нажатия на кнопку входа откроется окно авторизации в Telegram, автоматически созданное с помощью скрипта от Telegram Widget. Интерфейс данного окна представлен на рисунке 3.4.2.

 
Рисунок 3.4.2 – Интерфейс авторизации в Telegram
После успешного входа в аккаунт, приложение сохраняет токен доступа в локальном хранилище браузера и автоматически перенаправляет пользователя на главную страницу. Одновременно с этим чат-бот получает возможность отправлять сообщения пользователю.	
3.4.1.2 Главная web-страница приложения
	После авторизации пользователь попадает на основную страницу приложения. Интерфейс данной страница представлен на рисунке 3.4.3.
 
Рисунок 3.4.3 – Интерфейс главной web-страницы приложения
	В верхней части страницы находится шапка сайта, в левой части которой расположен логотип и название приложения, а в правой текущий пользователь и кнопка выхода из аккаунта, после нажатия на которую пользователя перенаправит обратно на страницу авторизации.
	В центральной части страницы располагается основная рабочая область, которая разделена на две вкладки: "Задачи" и "Статистика". Также есть кнопка перехода в чат-бота Telegram, чтобы пользователю не пришлось искать его вручную или дожидаться первого уведомления.
3.4.1.2.1 Вкладка «Задачи»
Данная вкладка выбирается по умолчанию при заходе на сайт или при обновлении страницы. Интерфейс вкладки представлен на рисунке 3.4.4.
 
Рисунок 3.4.4 – Интерфейс вкладки «Задачи»
	Вкладка «Задачи» делится на две области – панель управления задачами (область 1) и список задач (область 2).
	Панель управления задачами содержит:
-	кнопку добавления новой задачи;
-	фильтры для отображения необходимых задач (по категориям, по дате, по статусу выполнения);
-	кнопку настроек категорий;
-	кнопку экспорта задач в PDF.
Список задач содержит:
-	список активных задач с указанием текущего количества задач в нем;
-	список выполненных задач с указанием текущего количества задач в нем.
Вкладка «Задачи» позволяет осуществлять основной функционал приложения, а именно работать со списками своих дел.
После нажатия на кнопку «Добавить задачу» в начале списка активных задач появляется карточка-заготовка под новую задачу, где нужно обязательно ввести название задачи и выбрать категорию (или оставить выбранную по умолчанию). После ввода необходимых параметров задача останется в общем списке.
Различные параметры фильтрации позволяют отображать только необходимые пользователю задачи. Фильтры позволяют выбрать:
-	конкретную категорию (рисунок X – область Y);
-	конкретную дату (рисунок X – область Y);
-	сортировку по датам по определенному признаку (рисунок X – область Y);
-	статус задачи (рисунок X – область Y).
Можно выбирать несколько параметров, тем самым комбинируя их между собой.
Кнопка настроек категорий открывает модальное окно, где представлены все существующие категории и позволяет изменить их названия, цвет, удалить или добавить новые категории. Интерфейс окна настроек представлен на (рисунок X).
Кнопка экспорта в PDF также открывает модальное окно, где можно выбрать необходимые параметры экспорта, среди которых необходимые задачи (выбранные пользователем, все активные, все выполненные, за определенный срок), а также график продуктивности. После подтверждения создается и автоматически скачивается PDF файл, структура которого представлена на (рисунок X).
Сами задачи представляют собой блоки, содержащие:
-	чек-бокс для отметки о выполнении (рисунок X – область Y);
-	название (рисунок X – область Y);
-	описание – раскрывается по отдельной кнопку. А для его редактирования используется текстовый редактор Toast UI (рисунок X – область Y);
-	категорию со своим цветом (рисунок X – область Y);
-	кнопку настройки уведомлений (рисунок X – область Y);
-	кнопку редактирования (рисунок X – область Y);
-	кнопку удаления (рисунок X – область Y).
Также при работе с задачами существуют еще две удобные особенности:
-	возможность изменения порядка задач в списке активных путем их перемещения мышкой методом drag-and-drop;
-	если задача была перемещена в выполненные, но пользователь решил вновь вернуть ее в активные, то она вернется на свою прежнюю позицию.
	3.4.1.2.2 Вкладка «статистика»
	Вкладка «статистика» предоставляет пользователю визуальное представление о его задачах и успехах в их выполнении. Интерфейс данной вкладки представлен на рисунке (Рисунке X).
	Пользователю доступно текстовая информация, круговая диаграмма задач и график продуктивности.
	Диаграмма задач показывает четкую статистику по количеству задач в зависимости от выбранного типа. Всего есть три типа:
-	«по выполнению» – показывает количество выполненных и невыполненных задач (Рисунок X а);
-	«выполненные по категориям» – показывает количество задач в каждой категории из списка выполненных (Рисунок X б);;
-	«невыполненные по категориям» – показывает количество задач в каждой категории из списка активных (Рисунок X в).
График продуктивности имеет столбчатый вид и показывает количество выполненных задач по дням. При наведении на столбцы можно увидеть точное число выполненных в этот день задач. В графике учитывается именно дата выполнения, а не дата, написанная на самой задаче. Также можно выбрать период, за который будет построен график. Это может быть 7 дней, 14 дней, 30 дней или выбранный пользователем период от одной даты к другой.
